# Поддержка

Краткий путеводитель: куда обратиться за помощью по **CloudCastle DI**.

## Установка и использование

| Ресурс | Ссылка |
|--------|--------|
| Packagist | https://packagist.org/packages/cloudcastle/di |
| README и быстрый старт | [README.md](README.md) |
| Wiki (руководство) | [wiki/Home.md](wiki/Home.md) · [на GitHub](https://github.com/cloudcastle-apps/di/wiki/Home) |
| История изменений | [CHANGELOG.md](CHANGELOG.md) |

```bash
composer require cloudcastle/di
```

## Задать вопрос

| Канал | Когда использовать |
|-------|-------------------|
| [Discussions → Q&A](https://github.com/cloudcastle-apps/di/discussions/new?category=q-a) | Вопросы по API, интеграции, «как сделать…» |
| [Discussions → General](https://github.com/cloudcastle-apps/di/discussions/new?category=general) | Общие темы, обмен опытом |
| [Discussions → Ideas](https://github.com/cloudcastle-apps/di/discussions/new?category=ideas) | Предложения по улучшению API |

Перед вопросом укажите версию `cloudcastle/di`, PHP и минимальный пример кода.

## Безопасность

Уязвимости **не** публикуйте в Issues и Discussions. См. [SECURITY.md](SECURITY.md).

## Внести код

См. [CONTRIBUTING.md](CONTRIBUTING.md): `composer ci`, стиль PSR-12, Pull Request в `main`.

## Коммерческая поддержка

Проект распространяется «как есть» (MIT). Платная поддержка и консультации — через [GitHub Sponsors](https://github.com/sponsors/cloudcastle-apps) или Issues/Discussions для уточнения возможностей.

## Организация

Репозиторий поддерживается организацией [cloudcastle-apps](https://github.com/cloudcastle-apps).
